0 · 心法先行

为什么要学这个：你大概已经让 AI 帮你写过东西——问一句，它答一段，挺聪明。但这本笔记要带你去的地方不止于此：让 AI 真正帮你把一件事做完，比如把你一周的物理笔记整理成一个能发到网上的网页。要做到这一点，光会「问得巧」不够，你得先有一套心法：怎么跟它合作、怎么盯着它别跑偏。这一章不教任何命令，只把心法讲清楚——后面九章都是在反复练这套心法。心法比 prompt 技巧更重要，因为技巧会过时，心法不会。

大脑 vs 大脑＋手脚

先分清两个词，后面全靠它。

大语言模型（large language model / LLM）是「大脑」：你给它文字，它给你文字。它能想、能写、能解释，但它看不见也碰不到你电脑里的文件、打不开网页、跑不了任何程序。就像一个隔着电话给你出主意的聪明朋友——主意再好，手还是你自己动。

智能体（agent）是「大脑＋手脚」：同一个大脑，外加一套能真正动手的工具——读你的文件、改文件、运行命令。Anthropic 给 agent 的定义很朴素：让模型「在一个循环里自主地使用工具，根据环境的反馈一步步推进」（原文：LLMs ... using tools based on environmental feedback in a loop）。说人话——它不再只是回你一段话，而是动手做、看结果、再决定下一步，直到把事办成。

给模型这双「手」的机制，官方叫工具使用（tool use，也叫 function calling）：模型自己判断该用哪个工具，发出一次结构化的调用，由程序去执行，再把结果还给它。一句话记住区别：

🧠 心法回扣 · 一句话区别

LLM＝会思考的大脑；agent＝大脑＋一双能在你电脑里干活的手脚。这本笔记教的，就是怎么用好这双手脚、并且始终由你说了算。

为什么是「现在」才好用

agent 这个想法不新，但近一两年才真的顺手，靠三件事凑齐了：

  • 模型够聪明了。当前旗舰模型专门擅长复杂推理和「长链条」的自主任务（Anthropic 形容最强档 Claude 适合 complex reasoning, long-horizon agentic coding, and high-autonomy work）——能扛住一件事的好多个步骤而不乱。
  • 会用工具了。有了上面说的 tool use，模型能真正调用工具去动手；官方说「给 agent 工具是最有效的能力之一」，加上基础工具后在不少任务上提升巨大。
  • 记性够大了。现在的上下文窗口（context window）很大——大到能一次「读进」你整批笔记、甚至整个项目文件夹，它才谈得上跨多个文件帮你干活。

核心比喻：一个超聪明、但对你一无所知的实习生

🧠 心法回扣 · 把 agent 当成实习生

整本书你只要记住一个画面：你手下来了一个智商极高、但对你一无所知的实习生。

他聪明、肯干、动作快，可是——他不知道你是谁、在做什么、想要什么样的结果，也没见过你电脑里任何东西。所以你越是把背景交代清楚（你在干嘛、给他哪些材料、你心里「做好了」长什么样），他越能帮上忙；你越是甩一句「帮我弄一下」就走开，他越可能认认真真做出一个跑偏的东西。

带 agent 和带实习生是同一件事：讲清楚 → 让他做 → 你验收 → 给反馈让他改。

主线心法：四步闭环

这套「带实习生」的节奏，拆开就是贯穿全书的四步闭环。每次让 AI 干活，都在转这个圈：

① 交代 context
讲清背景、给齐材料
→
② 产出
让它动手做出东西
→
③ eval 验收
你来检查对不对
→
④ 修正
指出问题让它改

④ 之后带着新反馈回到 ① ，圈越转越准 ↻

eval（验收 / evaluation）就是「你这个产出到底行不行」的检查动作——这一步是你的核心职责，不能外包给 AI。agent 自己也靠环境的反馈推进（官方说它需要「来自环境的实地结果，比如工具调用结果或代码运行结果，来判断自己进展如何」），但「这东西是不是你真正想要的」，最终得你来拍板。

两个最常见的坑，和解药

⚠️ 坑一：被 AI 牵着走

AI 答得又快又像模像样，你很容易就一路点「好」「继续」，最后交出一个你自己都没真正看懂的东西。问题不在它做了什么，而在你跳过了 ③ 验收——方向盘不知不觉交了出去。

⚠️ 坑二：AI slop

AI slop（AI 糊弄货）指那种「看着挺像样、其实空洞没价值」的产物：废话连篇的长文、似是而非的解释、能跑但你不理解的代码。它通常是「① context 没交代清楚 ＋ ③ 没认真验收」一起酿成的。

🧠 心法回扣 · 同一副解药

两个坑，解药是同一个：你当那个清醒的验收者，把四步闭环转完整。交代清楚 context（少踩坑二），认真做 eval 和修正（少踩坑一）。AI 负责动手，你负责定义「什么叫做好」——这件事永远是你的。

三条主线，会一路陪你练

从下一章起，这本笔记会用三个真实例子反复带你练这套心法。它们会反复出现，越走越深：

  • ① 课堂笔记 → 网页（HTML）→ 发布到 GitHub Pages：把你的物理（Physics）笔记交给 AI 整理成一个网页，最后用 GitHub Pages 发到网上、拿到一个能分享的链接。这是贯穿全书的主线项目。
  • ② 第二大脑：用 Obsidian（一个把笔记存成本地文件的工具）配合 Claude，搭一个会越用越聪明的第二大脑（second brain），把学到的东西沉淀下来、随时调用。
  • ③ 机械工程兴趣拓展：顺着你对机械工程（mechanical engineering）的兴趣，让 AI 帮你查、帮你整理、帮你拓展，把「随便看看」变成真能积累的探索。

这本笔记主要用 Claude Code（Anthropic 的命令行 agent 工具，官方定位：an agentic coding tool that reads your codebase, edits files, runs commands，住在你的终端里）。市面上同类工具还有 Gemini CLI（Google）、Codex CLI（OpenAI）等——它们路子相通，留到第 8 章再触类旁通，现在你只要知道「不止一家」就够了。

小结

  • LLM＝大脑（只产文字）；agent＝大脑＋手脚（在循环里用工具动手做事）。
  • 把 agent 当成超聪明但对你一无所知的实习生：你交代得越清楚，它越靠谱。
  • 主线心法是四步闭环：交代 context → 产出 → eval 验收 → 修正，转完再回到起点。
  • 两个坑——被 AI 牵着走、AI slop——同一副解药：你当清醒的验收者，定义「什么叫做好」。
  • 三条主线（笔记→网页、第二大脑、机械兴趣）会一路陪你把心法练熟。

一句话回顾：AI 是你那个超聪明的实习生，心法就是「交代清楚 → 让它做 → 你验收 → 让它改」；技巧会过时，这套心法不会。

1 · 三层界面各管什么

为什么要学这个：你已经知道 LLM 是"大脑"、agent 是"大脑＋手脚"——但具体在哪里用哪个？打开 claude.ai 对话窗口就够了，还是得开命令行？或者还有什么中间选项？这一章把三层界面说清楚，给你一个真正能用的决策框架：用完即走就留在网页；想让 agent 碰你的本地文件、但还没准备好命令行，有 Claude Cowork 接着；要长期维护项目、要发布上线、要完全掌控，再毕业到 Claude Code。搞清楚这三层，你后面不会走弯路。

先尝一口甜头：在网页里整理一段笔记

先别管界面怎么选，直接动手试一次——用 claude.ai（Anthropic 官方的网页 / 桌面 App）整理你的 Physics 课堂笔记。不需要装任何东西，打开浏览器就能做。

🙌 跟我做 · 示例① 起点：整理一段 2026-week3 笔记

1. 在浏览器里打开 claude.ai，登录你的账号。

2. 开一个新对话，把下面这段话（换成你自己这周的笔记内容）贴进去：

我是高中物理学生，下面是我 2026-week3 课堂的一段乱糟糟笔记，
帮我整理成干净的中英双语 HTML 格式，要有标题、要点、公式用 <em> 标出。
笔记：[把你的笔记粘在这里]

3. 发送，看 Claude 产出整理好的 HTML。

4. 试着问它"把第一条要点再解释得通俗一点"，练一次修正（四步闭环的 ④）。

这就是主线示例①「课堂笔记 → 网页（HTML）→ 发布到 GitHub Pages」的第一脚——你已经拿到了 AI 整理的 HTML 片段。

感受到了吗？交代 context（给笔记）→ 产出（Claude 整理）→ eval 验收（你看一眼）→ 修正（再问一句），四步闭环转了一圈，一分钟内。

撞墙：结果留不下来，也碰不到文件

现在问题来了：那段整理好的 HTML 只存在这一个对话里。你关掉页面、下周再想用，得重新来过。更关键的是——它碰不到你电脑里的文件。你没法让 claude.ai 直接帮你把整理好的内容写进你桌面上的 2026-week3.md，更别说帮你建整个文件夹结构、或者一次处理十几周的笔记。

这不是 Bug，这就是 claude.ai 网页/App 的设计边界：它的产出存在对话里，不在你的硬盘里（能接收你上传的文件分析，但分析完结果只在对话里，不会回写到硬盘）。要让 agent 真正动手碰你的本地文件，需要往下走一层——这时候就出现了两条路：一条零命令行，一条完全掌控。

决策框架：三层进阶模型

不是谁比谁好，是场景不同、能力不同。下面这张表帮你一眼定位：

层级 能做什么 碰不到 / 做不到 最适合的场景
第 1 层
claude.ai 网页 / App		 Research（联网多步研究）（需付费档）；Artifacts（交互可视化 / 迷你页）；上传文件快速分析；轻量问答探索 读写你硬盘上的文件；跨会话持久维护项目 用完即走：临时查资料、头脑风暴、整理一段笔记、生成可分享的交互小页
第 2 层
Claude Cowork		 零 terminal、不写代码，直接读写本地文件 / 文件夹；和 Claude Code 同一套 agentic 引擎；在 Claude 桌面 App 里启动 跑自定义命令；版本控制（git）；自动发布到 GitHub Pages；编程级自动化 想让 agent 整理 / 改名 / 归档你的本地文件夹，但还没准备好命令行
第 3 层
Claude Code（CLI）		 读写文件、跑任意命令、git 版本控制、批量自动化、发布到 GitHub Pages；完全可编程；长期维护项目 （没有硬边界——这一层你全拿得到） 要长期维护、要发布上线、要完全掌控：主线示例①「课堂笔记 → 网页（HTML）→ 发布到 GitHub Pages」的完整路线就在这一层完成

⚠️ Cowork 的平台限制

Claude Cowork 目前只在 Claude 桌面 App（macOS / Windows） 里运行，不支持网页端或手机端。访问需要付费档（Pro、Max、Team、Enterprise）。（来源：Anthropic 官方支持文档，2026-06-23 核实。）

Cowork 和 Claude Code 是什么关系？

官方说得直接：Claude Cowork 用的是「和 Claude Code 同一套 agentic 架构（same agentic architecture that powers Claude Code）」，只是入口搬进了 Claude 桌面 App 的 GUI 界面，不需要打开 terminal。两者的区别在于控制深度——Cowork 给你"零 terminal 动文件"的入门；Claude Code 给你"跑命令、版本控制、自动发布"的完全掌控。Cowork 不取代 Claude Code，而是它的无 terminal 入门版。

换个比喻：Cowork 像自动挡，Claude Code 像手动挡。自动挡先把你开上路，手动挡让你完全控制车子——路是同一条，发动机也是同一颗。

本地可视化：第 2 / 3 层都能做

你可能想："网页端的 Artifacts（交互可视化产物）挺酷的，本地能不能做？"能——在第 2 层让 Cowork 帮你生成一个 HTML 可视化文件、直接存到你的文件夹；或者到了第 3 层，让 Claude Code 写 Python 脚本画图，跑完结果留在你机器上、纳入版本控制。本地可视化的好处是产物是你的文件，随时能改，不依赖任何在线服务。

🧠 一句话原则

用完即走的探索 → 网页 / App（第 1 层）；想动本地文件又不想碰命令行 → Claude Cowork（第 2 层）；长期维护 / 发布 / 完全掌控 → Claude Code ＋ 备份（第 3 层）。

记不住表没关系，记住这句话。每次启动 AI 前先问自己两个问题："这件事做完就完了，还是我以后还要改？要不要碰我硬盘上的文件？" 答案给了，层级也给了。

心法在三层下完全一致

🧠 心法回扣 · 换了界面，心法不换

无论你在 claude.ai 网页里整理笔记、在 Cowork 里整理本地文件夹、还是在 Claude Code 里批量处理十几周的 Physics 笔记——四步闭环不变：交代清楚 context → 让它产出 → 你来 eval 验收 → 给反馈修正。

界面是工具，心法才是本事。工具换了心法还在，你永远是那个清醒的验收者。

小结

  • 第 1 层 · claude.ai 网页 / App：用完即走，适合联网研究（Research）、交互可视化（Artifacts）、轻量一次性探索。碰不到你的本地文件。
  • 第 2 层 · Claude Cowork：零 terminal、不写代码，却能直接读写本地文件 / 文件夹；跑在 Claude 桌面 App 里，和 Claude Code 同一套 agentic 引擎；付费档可用，仅限桌面（macOS / Windows）。是非技术者让 agent 动本地文件的最丝滑入门。
  • 第 3 层 · Claude Code（CLI）：完全掌控——读写文件、跑命令、版本控制、批量自动化、发布到 GitHub Pages。主线示例①「课堂笔记 → 网页（HTML）→ 发布到 GitHub Pages」的完整路线，在这一层完成。
  • Cowork 不取代 Claude Code，是它的无 terminal 入门版；terminal 与 Claude Code 是完整教学目标——是你明确要学的，定位成毕业到更强、更可控的一层。
  • 心法在三层下一致：四步闭环在哪里都管用，你永远是那个说了算的验收者。

一句话回顾：用完即走找网页（第 1 层）；想动本地文件又怕命令行找 Cowork（第 2 层）；要长期维护 / 发布 / 完全掌控，毕业到 Claude Code（第 3 层）——下一章先去认识命令行。

2 · terminal 与文件系统

为什么要学这个：上一章说了，第 2 层 Claude Cowork 能让你零命令行就动本地文件——那为什么还要学 terminal？因为要真正毕业到第 3 层 Claude Code——让 agent 帮你跑命令、管版本、发布到 GitHub Pages、做可以长期维护的项目——你就必须认识 terminal。光这一点就值得学。更有用的是：当你在 Claude Code 里看着 agent 自己打命令、翻文件夹，你会真正看懂它在干什么、哪步可能出问题、该怎么纠偏。这一章不要求你变成命令行高手，只要你读完之后，能在 terminal 里找到自己在哪、能看文件、能建文件夹——再加上后面让 Claude Code 带你跑，就够用了。

命令行 vs 图形界面：一个类比

你平时用 Mac 管理文件，靠的是图形界面（Graphical User Interface / GUI）：双击文件夹、把文件拖来拖去、右键重命名。这就像在餐厅指着菜单说"我要这个"——直觉、方便，但菜单只给你有限的选项。

命令行（command line），或者叫终端（terminal），则像直接跟大厨说话："帮我把 A 文件夹里所有以 2026 开头的 .md 文件，全部复制到 B 文件夹，然后把名字改掉。" 更费脑力，但能做点菜单里根本没有的事，而且——可以让 agent 帮你打那些字。

对你来说，terminal 最大的价值不是你自己背一堆命令，而是：看懂 agent 在做什么，在它做错时你能叫停、纠正。

打开 Terminal.app

macOS 自带了一个 Terminal（终端）应用，就叫 Terminal.app，不需要安装任何东西。

🙌 跟我做 · 打开你的第一个 terminal 窗口

1. 点击 Dock 里的 Launchpad 图标（像火箭的那个）。

2. 在搜索栏输入 Terminal，然后点击出现的 Terminal 图标。

（另一种方法：Finder → Applications → Utilities → Terminal.app 双击打开。）

你会看到一个黑色或白色的小窗口，最后一行有个闪烁的光标，大概长这样：

Last login: Mon Jun 23 ...
你的用户名@你的电脑名 ~ %

那个 % 符号是命令提示符（prompt），它说："好了，等你输命令。" 你输入的所有命令都会在这里打，然后按 Return 运行。

macOS 从 Catalina 起默认用的 shell 叫 zsh（读作"zee shell"），这也是 Claude Code 运行的环境。提示符里看到 % 就是 zsh，看到 $ 是 bash——你的应该是 %。（来源：Apple 官方 Terminal 文档，2026-06-23 核实。）

路径：你在文件系统里的"地址"

在 Finder 里，你靠视觉看到文件夹套文件夹；在 terminal 里，每个位置都有一个文字地址，叫路径（path）。

路径有两种：

  • 绝对路径（absolute path）：从根目录 / 出发，写出完整地址。如 /Users/你的用户名/study/physics-notes/——就像写"新加坡市裕廊东 123 号"，永远不会搞错。
  • 相对路径（relative path）：从当前位置出发，只写剩余部分。如你已经在 /Users/你的用户名/，那 study/physics-notes/ 就够了——就像跟已经在同楼的人说"三楼左转"，比写完整地址省事。

三个特殊符号要记住：

  • ~（tilde 波浪号）：你的家目录（home directory）的快捷方式，等价于 /Users/你的用户名/。你的文件大多放在这里的子文件夹里。
  • .（一个点）：当前目录本身。
  • ..（两个点）：上一级目录（父目录）。用 cd .. 往上退一层。

第一批命令：文件浏览

下面逐个认识最常用的命令，每个都有一个小练习——请在你自己的 terminal 里跟着做。

🙌 跟我做 · pwd：我在哪里？

pwd（Print Working Directory）告诉你当前在文件系统里的哪个位置。

pwd

预期输出（你的用户名会不一样）：

/Users/你的用户名

刚打开 terminal，你通常就在家目录（~）。看到这个路径就对了。

🙌 跟我做 · ls：这里有什么？

ls（List）列出当前目录里的文件和文件夹。

ls

预期输出（你看到的名字会不一样）：

Desktop    Documents    Downloads    Movies    Music    Pictures

试试加 -l 选项，看更多细节（文件大小、修改时间）：

ls -l

输出会变成每行一个文件，有日期和大小。加 -la 还会显示以 . 开头的隐藏文件（如 .zshrc）。

🙌 跟我做 · cd：走过去

cd（Change Directory）切换到另一个目录。

cd Documents

没有任何输出就对了——terminal 的风格是"没问题就沉默"。然后用 pwd 确认你已经进去了：

pwd

预期输出：

/Users/你的用户名/Documents

往回退一层：

cd ..

回到家目录（随时可用）：

cd ~

🙌 跟我做 · mkdir：新建文件夹

mkdir（Make Directory）新建一个文件夹。我们来建主线示例①的工作文件夹：

mkdir -p ~/study/physics-notes/raw

（-p 表示如果中间的父文件夹也不存在，一起建好，不报错。）

验证是否建好：

ls ~/study

预期输出：

physics-notes

你刚建好了主线项目的工作目录 ~/study/physics-notes/——Claude Code 之后就在这里帮你整理 Physics 笔记。

🙌 跟我做 · cat：看文件内容

cat（Concatenate and print）把文件内容打印到 terminal 里。先建一个测试文件（不用担心这步的语法，先照做即可）：

echo "Newton's second law: F = ma" > ~/study/physics-notes/raw/2026-week3.md

然后用 cat 看它：

cat ~/study/physics-notes/raw/2026-week3.md

预期输出：

Newton's second law: F = ma

这就是你的主线笔记文件的起点——之后会交给 Claude Code 整理。

🙌 跟我做 · cp：复制文件

cp（Copy）复制文件到另一个位置。

cp ~/study/physics-notes/raw/2026-week3.md ~/study/physics-notes/raw/2026-week3-backup.md

验证：

ls ~/study/physics-notes/raw

预期输出：

2026-week3-backup.md    2026-week3.md

复制整个文件夹要加 -r（recursive 递归）：cp -r 源文件夹 目标位置。

🙌 跟我做 · mv：移动或重命名

mv（Move）既可以移动文件，也可以重命名——在同一个文件夹里"移动到新名字"就是重命名。

mv ~/study/physics-notes/raw/2026-week3-backup.md ~/study/physics-notes/raw/2026-week3-bak.md

验证（文件名已变）：

ls ~/study/physics-notes/raw

预期输出：

2026-week3-bak.md    2026-week3.md

🙌 跟我做 · open：用 Finder 打开

open 是 macOS 专有命令，能在 Finder 里打开文件夹，或者用默认 App 打开文件。

open ~/study/physics-notes

预期结果：Finder 跳出来，打开到 physics-notes 文件夹——你能看到刚才建的 raw/ 子文件夹。

特别实用的用法：open .（在 Finder 里打开当前目录）——当你在 terminal 里工作、想用 Finder 看一眼同一个地方时，就用这个。

⚠️ 安全盒：rm 是永久删除，没有废纸篓

rm（Remove）是最危险的常用命令。它直接永久删除文件，不会移到废纸篓（Trash），删了就删了，除非你恰好有 Time Machine 备份。

Finder 里把文件拖到废纸篓还有机会找回来；rm 没有。

用 rm 之前，养成一个习惯：先 ls 看清当前目录，用 pwd 确认你在哪里，再运行 rm。

# 删单个文件（只是示意，别把你需要的文件删掉）
rm ~/study/physics-notes/raw/2026-week3-bak.md

尤其要避免加 -rf 组合选项（强制、递归删整个文件夹）。一旦对着错误的路径跑，整个文件夹瞬间消失，无法撤销。Claude Code 有时会用 rm——这正是为什么你需要在旁边读懂它在做什么，而不是闭眼说"好"。

Terminal.app 补充：不必是你唯一的 terminal

macOS 自带的 Terminal.app 够用，但很多开发者偏爱 iTerm2（免费，功能更多）或者 VS Code 内建的 terminal。选哪个不影响命令，命令都是一样的。入门阶段用 Terminal.app 最省事，之后换不换随你。

Homebrew：terminal 里的"App Store"

Homebrew 是 macOS 上最流行的包管理器（package manager），用来在 terminal 里安装各种命令行工具——比如 git（版本管理）、node（跑 JavaScript）、python 等。官方介绍它是 "The Missing Package Manager for macOS"。以后用 Claude Code 时，它可能让你先装一些工具；知道 Homebrew 是什么，你就不会慌。

官方安装命令（来源：brew.sh，2026-06-23 WebFetch 核实）：

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

⚠️ 安装 Homebrew 前先确认

从网上下载脚本直接运行，风险不低。请确认：① 你是从官方网站 brew.sh 复制的命令；② 你的网络可信。如果 Claude Code 之后提示你安装某个工具，它通常会帮你生成对应的 brew install ... 命令，你确认后再跑就好。

装好 Homebrew 之后，安装工具的命令格式是 brew install 工具名，例如 brew install git。

🧠 心法回扣 · 你不必记住所有命令

这章的命令你不需要全背。真正的工作流是：你大概知道有哪些命令、它们能做什么 → Claude Code 帮你打出具体命令 → 你读懂、确认、按 Enter。 你的角色是"懂行的验收者"，不是"人肉搜索引擎背命令"。遇到不认识的命令，随时可以问 Claude。

小结

  • Terminal.app 是 macOS 自带的命令行窗口；打开方法：Launchpad → 搜索 Terminal。默认 shell 是 zsh（提示符 %）。
  • 路径是文件在系统里的"地址"：绝对路径从 / 开始；相对路径从当前位置算。~ 是家目录，. 是当前目录，.. 是上一级。
  • 核心浏览命令：pwd（在哪里）、ls（看什么）、cd（去哪里）、cat（看文件内容）、open .（在 Finder 里打开）。
  • 文件操作：mkdir（建文件夹）、cp（复制）、mv（移动 / 重命名）。
  • rm 永久删除，没有废纸篓，跑前先 ls + pwd 确认。
  • Homebrew 是 macOS 的包管理器，用来安装命令行工具；安装命令从 brew.sh 官网复制。
  • 你不需要背命令——懂原理、看得懂 agent 在做什么，才是重点。

一句话回顾：terminal 是你和 Claude Code 共同工作的地盘——学会"看懂它在哪、看懂它在做什么"，比背命令更重要；下一章直接开进 Claude Code。
