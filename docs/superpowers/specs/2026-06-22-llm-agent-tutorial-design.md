# 设计文档：LLM → Agent 教学笔记（给弟弟的中文 HTML 笔记）

日期：2026-06-22
状态：已通过 brainstorm 确认（第二轮 · 心法为轴版），待写实现计划

## 1. 项目目标

为一位**会用电脑但非技术**的受众（用户的弟弟）制作一套**中文为主、穿插英文术语**的 HTML 教学笔记，
讲清楚"LLM 向 Agent 转化"这波浪潮、由此产出的 CLI agent 工具（以 Claude Code 为主轴），
**以及——最重要的——如何与 AI 高质量协作并把它真正用进自己的学习和生活。**

核心定位（本轮关键决定）：**心法为轴，工具为用。**
不是一套"工具操作手册"，而是一套"如何与 AI 协作来改造学习/生活"的指南，terminal、Claude Code、MCP 等
都是为这条主线服务的手段。

读完后受众应能：
1. 理解与 AI 协作的核心心法（交代清楚 context → 产出 → eval 验收 → 修正的闭环），以及它为什么比 prompt 技巧更重要；
2. 判断什么时候用网页/App 够了、什么时候必须上本地 CLI agent；
3. 在 macOS 上用 terminal 完成基本文件操作；
4. 安装并运行 Claude Code，让它在一个真实项目里帮自己干活；
5. 把产出沉淀成 Markdown/HTML、托管到 GitHub Pages，并理解"第二大脑"知识库的玩法；
6. 大致理解 MCP、Skills 等扩展能力，以及其他 harness 与 Claude Code 的关系。

## 2. 受众画像

- 用户的弟弟，**17 岁**，在**新加坡公教中学（CHS）全奖**读高中，**明年参加 O-level** 考试升 JC。
- 从国内读完高一后降级转学过去，课程内容对他没有压力；学校允许带电脑。
- 会日常使用 macOS（图形界面没问题），但**几乎没用过命令行**、不熟悉文件路径概念。
- 非技术背景，需要从最基础讲起，语气友好、循序渐进、多用生活化类比。
- **兴趣点：机械工程（mechanical engineering）。** 内容应针对他的日常与兴趣 tailored。
- 语言：**中文讲解为主、穿插英文术语**——对他阅读效率最高，也方便他之后就某个术语自行 research。

## 4. 贯穿全篇的脊柱（spine）

整套笔记由一条心法主线、一个核心比喻、三个咬合的真实示例串起来。

### 4.1 核心心法（主线）

`交代清楚 context → 让它产出 → 你来 eval 验收 → 反馈修正` 的**闭环**。

立论：随着 LLM 能力越来越强，与它协作的瓶颈不再是"会不会下指令 / prompt 技巧"，
而是**你能否把自己的诉求、自己和项目的现状，作为高质量 context 交代清楚**，
以及**对产出有没有足够好的 eval**。这两件事正是避免"被 AI 牵着走"和"产出 AI slop"的解药。

### 4.2 核心比喻

把 AI 当一个**"超聪明、但刚来你身边、对你一无所知的实习生"**：
你越能讲清你是谁、要干嘛、好的标准长什么样，它越有用；交活时你得验收，而不是照单全收。

### 4.3 三个主线示例（每学一个新概念就回到它们推进）

1. 📝 **课堂笔记 → 整理成 HTML → GitHub Pages** 随时随地复习。
2. 🧠 **第二大脑知识库（Obsidian + Claude）**——前一个示例的沉淀，以及机械兴趣的探索，都汇流到这里。
3. ⚙️ **机械工程兴趣拓展**——让整套笔记对他"有趣"而不只"有用"。

## 5. 内容结构（章节走向 0–9）

### 第 0 章 · 心法先行
- LLM → agent 一句话区别：LLM = 会思考的大脑；agent = 大脑 + 手脚（读写文件、跑命令、用工具）。
- "会聊天 → 会动手"的演变；为什么是现在（模型能力 + 工具调用 + 长上下文）。
- 核心比喻：超聪明但对你一无所知的实习生。
- 核心心法：`context → 产出 → eval → 修正` 闭环；为什么它比 prompt 技巧更重要。
- 两个坑及解药：被 AI 牵着走 / 产出 AI slop。
- 引出三个会贯穿全篇的主线示例。

### 第 1 章 · 两种界面各管什么：网页/App vs 命令行 agent（本轮加重）
- 先用 **Claude 网页/桌面 App** 在"整理一段课堂笔记"上跑通心法、立刻尝到甜头（无需 terminal）。
- 随即撞墙：产出留不下来、碰不到你的文件 → 引出为什么需要能碰文件的 CLI agent。
- **判断框架 / 决策原则**：
  - **网页/App 更优**：Deep Research（多源调研带引用）、实时可视化 / Artifacts、用完即走的轻量探索。
  - **CLI（Claude Code）更优**：任何想长期维护的项目（本地建文件夹 + CLI，进度留本地 + 云盘/GitHub 备份）、
    需要碰真实文件 / 跨会话续上下文 / 批量处理的活；本地可视化也可让 agent 写 **Python 或 HTML** 出图。
  - 一句话原则：**用完即走的探索 → 网页；要沉淀、要维护、要碰文件 → 本地 + CLI + 备份。**
- （网页端功能的真实名称与形态落笔前 research 核实。）

### 第 2 章 · 前置：terminal 与文件系统（非技术友好完整入门，份量较重）
- 什么是命令行 / terminal，与图形界面的区别，为什么 agent 偏爱它。
- 文件系统与路径：绝对 vs 相对路径，`~` `.` `..` 含义。
- 常用命令逐个举例 + 🙌 跟我做小练习：`pwd` `ls` `cd` `cat` `mkdir` `cp` `mv` `rm` `open`（以文件浏览为主）。
- 安全意识（`rm` 的危险、看清路径）。
- macOS 特有：Terminal.app / iTerm 简介、Homebrew 简介。

### 第 3 章 · 上手 Claude Code
- 核心概念：模型是大脑，**harness** 是给它装手脚的工作台（工具 / 循环 / 权限 / 上下文管理）。
- 安装 Claude Code（前置条件、账号或 API key、**成本说明 + 学生可行选项**——research 核实当前方式）。
- 第一次运行：进项目目录 → 对话 → 观察它读文件 / 改文件 / 求权限。
- 心智模型：一个"读 → 想 → 动手 → 再读"的循环。
- 跑通主线示例①的第一段（课堂笔记进 workspace）。

### 第 4 章 · 把心法用熟：context 与 eval 的实战（灵魂章）
- 怎么喂高质量 context：CLAUDE.md、项目 workspace、把你的现状 / profile / 诉求交代清楚。
- 怎么做 eval：review 它的改动、不照单全收、把验收反馈喂回去迭代。
- 权限模式与安全、斜杠命令 `/`、计划模式、长任务怎么拆——**全部挂在"怎么交代清楚 / 怎么验收"这条线上**。
- 继续推进主线示例。

### 第 5 章 · 让产出沉淀：Markdown、HTML 与 GitHub Pages
- 为什么 **Markdown / HTML** 是 AI 时代最通用、最高效的文件/文本格式。
- 走完主线示例①：笔记 → HTML → GitHub Pages，随时随地复习。

### 第 6 章 · 第二大脑：跟 AI 共建知识库
- Karpathy 提出的"第二大脑"概念（research 核实原始出处与表述）。
- Obsidian + Claude 怎么搭：把笔记、想法、查过的东西沉淀成可检索、会生长的知识库。
- 主线示例②③在这里汇流（笔记沉淀、机械兴趣探索都进同一个库）。

### 第 7 章 · 扩展能力：MCP 与 Skills（有意做轻）
- MCP：给 agent 接外部工具 / 数据源的"统一插头"协议。

- subagents、hooks 一句话简介。
- 理念：**按需添加**，别一上来堆一堆。

## 6. 贯穿全篇的固定教学元素

- 每章开头一句"**为什么要学这个**"。
- 大量生活化类比（实习生、统一插头、工作台）。
- 🙌 **跟我做**步骤盒（命令 + 预期看到的结果）。
- ⚠️ **注意 / 安全**提示盒。
- 🧠 **心法回扣**小标记——每次某个操作体现了"context/eval 闭环"时点一下，强化主线。
- 每章结尾"**小结 + 一句话回顾**"。

## 7. HTML 形式

形式跟随内容——内容骨架成型后最终敲定。

- 倾向方案：**带左侧目录导航的静态站点**（多页或单长页 + 侧边栏），干净、便于反复查阅、
  纯静态、一个文件夹即可在浏览器打开，无需构建工具或联网。
- 视觉：简洁、易读、中文友好排版；代码 / 命令用等宽字体高亮；"跟我做""注意""心法回扣"盒子有明显视觉样式。
- 最终形式（多页 vs 单长页）在实现计划阶段、内容结构确定后再定。

## 8. 工作流程（两步走）

1. **Research 阶段**：逐章上网核实当前细节（官方文档 + 代码仓库 + release notes），整理出每章的事实素材与来源。
2. **落地阶段**：据 research 结果填充内容、生成 HTML。

## 9. 明确不做（YAGNI）

- 不做需要构建工具 / 服务器 / 联网的复杂前端；保持纯静态。
- 不做多 harness 平均用力的全景百科；以 Claude Code 为主、其他触类旁通（第 7、8 章有意做轻）。
- 不深入受众用不到的高级开发者话题（除非作为"进阶可选"一笔带过）。
- 不教 prompt 奇技淫巧；重心放在"交代清楚 context + 做好 eval"。
