# 设计文档：LLM → Agent 教学笔记（给弟弟的中文 HTML 笔记）

日期：2026-06-22
状态：已通过 brainstorm 确认，待写实现计划

## 1. 项目目标

为一位**会用电脑但非技术**的受众（用户的弟弟）制作一套**中文 HTML 教学笔记**，
讲清楚"LLM 向 Agent 转化"这波浪潮，以及由此产出的 CLI agent 工具（以 Claude Code 为主轴）。

核心目标：**先懂原理、再能上手**。
读完后受众应能：
1. 理解 LLM 与 agent 的区别、这波 CLI 工具浪潮是什么、为什么是现在；
2. 在 macOS 上用 terminal 完成基本文件操作；
3. 安装并运行 Claude Code，让它在一个项目里帮自己干点活；
4. 大致理解 MCP、Skills 等扩展能力，以及其他 harness 与 Claude Code 的关系。

## 2. 受众画像

- 会日常使用 macOS（图形界面没问题），但**几乎没用过命令行**、不熟悉文件路径概念。
- 非技术背景，需要从最基础讲起，语气友好、循序渐进、多用生活化类比。
- 学习偏好：先理解原理，再动手实践（平衡型）。

## 3. 最高优先级原则：研究严谨性（铁律）

**填充任何章节内容前，必须充分上网 research，以官方文档和代码仓库为准。**

- 原因：这些工具（Claude Code、Codex CLI、MCP、Skills 等）迭代极快，凭记忆写必然产生
  过时或事实性错误（版本号、命令、安装方式、成本、工具真实名称）。
- 要求：
  - 每章/每节先 research、后落笔；关键事实标注来源（官方 docs / GitHub repo）。
  - 命令、版本、定价、工具名一律不得凭记忆填写。
  - 核实用户给出的不确定名称（如 "openclaw"、"hermes"）对应的真实工具后再使用。
  - 优先信源顺序：① 官方文档 → ② 官方 GitHub 仓库 / release notes → ③ 权威二手资料。

## 4. 内容结构（学习路径）

### 第 0 章 · 为什么会有这波浪潮（背景）
- 从"会聊天的 ChatGPT"到"会动手的 agent"的演变。
- 一句话区别：LLM = 会思考的大脑；agent = 大脑 + 手脚（读写文件、跑命令、用工具）。
- 为什么是现在：模型能力提升 + 工具调用 + 长上下文。
- 一张全景图：当前主流 CLI agent 工具有谁。

### 第 1 章 · 前置知识：terminal 与文件系统（**完整入门教程，份量最重**）
- 什么是命令行 / terminal，与图形界面的区别，为什么 agent 偏爱它。
- 文件系统与路径：绝对路径 vs 相对路径，`~` `.` `..` 含义。
- 常用命令逐个举例 + 小练习：`pwd` `ls` `cd` `cat` `mkdir` `cp` `mv` `rm` `open`（以文件浏览为主）。
- 安全意识（`rm` 的危险、看清路径）。
- macOS 特有：Terminal.app / iTerm 简介、Homebrew 简介。

### 第 2 章 · Agent harness 是什么 + Claude Code 上手
- 核心概念：模型是大脑，**harness** 是给它装手脚的外壳（工具 / 循环 / 权限 / 上下文管理）。
- 安装 Claude Code（前置条件、账号或 API key、成本说明——research 核实当前方式）。
- 第一次运行：进项目目录 → 对话 → 观察它读文件 / 改文件 / 求权限。
- 心智模型：一个"读 → 想 → 动手 → 再读"的循环。

### 第 3 章 · Claude Code 用法与技巧（重点展开）
- 权限模式与安全。
- 斜杠命令 `/`、CLAUDE.md（给项目立规矩 / 记忆）。
- 上下文管理、计划模式、长任务怎么拆。
- 实战技巧：怎么提问、怎么喂上下文、怎么 review 它的改动。

### 第 4 章 · 扩展能力：MCP 与 Skills
- MCP：给 agent 接外部工具 / 数据源的"统一插头"协议。
- Skills：给 agent 打包的"专长 / 流程"。
- subagents、hooks 一句话简介。
- 理念：按需添加，别一上来堆一堆。

### 第 5 章 · 触类旁通：其他 harness
- 因为底层相通，快速过：Codex CLI、Gemini CLI、Goose、Aider、OpenCode 等（以 research 核实的当前列表为准）。
- 一张对比表：各自定位 + 与 Claude Code 的异同。
- 核实并澄清用户提到的 "openclaw"、"hermes" 实际对应的工具。

## 5. 贯穿全篇的固定教学元素

- 每章开头一句"为什么要学这个"。
- 大量生活化类比（agent = 给大脑装手脚；MCP = 统一插头；harness = 给大脑配的工作台）。
- 重点动手处放 **"🙌 跟我做"** 步骤盒子（命令 + 预期看到的结果）。
- 关键处放 **"⚠️ 注意 / 安全"** 提示盒。
- 每章结尾"小结 + 一句话回顾"。

## 6. HTML 形式

形式跟随内容——内容骨架成型后最终敲定。

- 倾向方案：**带左侧目录导航的静态站点**（多页或单长页 + 侧边栏），干净、便于反复查阅、
  纯静态、一个文件夹即可在浏览器打开，无需构建工具或联网。
- 视觉：简洁、易读、中文友好排版；代码 / 命令用等宽字体高亮；"跟我做""注意"盒子有明显视觉样式。
- 最终形式在实现计划阶段、内容结构确定后再定。

## 7. 工作流程（两步走）

1. **Research 阶段**：逐章上网核实当前细节（官方文档 + 代码仓库），整理出每章的事实素材与来源。
2. **落地阶段**：据 research 结果填充内容、生成 HTML。

## 8. 明确不做（YAGNI）

- 不做需要构建工具 / 服务器 / 联网的复杂前端；保持纯静态。
- 不做多 harness 平均用力的全景百科；以 Claude Code 为主、其他触类旁通。
- 不深入受众用不到的高级开发者话题（除非作为"进阶可选"一笔带过）。

## 9. 待实现计划阶段确认的开放项

- HTML 最终形式（多页 vs 单长页）。
- 是否需要可运行的练习沙盒目录供受众练手。
- 章节最终顺序微调（如背景章是否拆分）。
